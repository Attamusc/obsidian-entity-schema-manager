## Contributing to Documentation

When updating documentation:

1. **Keep synchronized**: Update cross-references when adding new sections
2. **Link properly**: Use relative paths for internal documentation links  
3. **Update INDEX.md**: Add new sections to the master index
4. **Test examples**: Verify all code examples and configurations work
5. **Maintain structure**: Follow the established document hierarchy

## Documentation Standards

- **Markdown format**: All documentation uses GitHub-flavored Markdown
- **Code examples**: Include working TypeScript/JSON examples where relevant
- **Cross-references**: Link between documents using relative paths
- **Up-to-date**: Keep documentation synchronized with code changes
- **Accessible**: Write for both end-users and developers
